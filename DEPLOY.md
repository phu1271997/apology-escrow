# Deployment Guide — ApologyEscrow on GenLayer

This guide provides step-by-step instructions for deploying and testing the **ApologyEscrow** smart contracts on the GenLayer Studio.

---

## Pre-Deploy Checklist
Before deploying, ensure both contract files conform to these strict validation rules:

- [ ] **Line 1 & 2 Comments**:
  ```python
  # v0.2.16
  # { "Depends": "py-genlayer:1jb45aa8ynh2a9c9xn3b7qqh8sm5q93hwfp7jqmwsfhh8jpz09h6" }
  from genlayer import *
  ```
- [ ] **No TreeMap/DynArray Reassignment**: Verify that `__init__` does not reassign storage types (e.g. `self.offender = TreeMap()`). They auto-initialize empty. Only initialize scalars in `__init__`.
- [ ] **No Floats**: Verify that no method signatures or state variables utilize Python `float`. Use `int` or sized ints like `u256`.
- [ ] **Allowed Types only in Signatures**: Verify that all parameters and return types of public methods are within: `str`, `bool`, `bytes`, `int`, sized ints, `Address`, `DynArray[T]`, and `TreeMap[K,V]`. Any dictionary or list must be serialized to a JSON string using `json.dumps`.
- [ ] **Nondet wrapping**: Verify that all `gl.nondet.*` methods are called inside a function passed to `gl.vm.run_nondet_unsafe`.
- [ ] **Consensus Stability**: Verify that the validator checks agreement on the `verdict` bucket (`QUALIFIES` or `FAILS`) rather than exact scores or reasoning text.

---

## Deployment Steps

1. **Access GenLayer Studio**:
   Open [GenLayer Studio](https://studio.genlayer.com/run-debug) in your browser.
2. **Clear Environment State**:
   - Go to **Settings** in the bottom left.
   - Click **Reset Storage** and confirm.
   - Perform a hard refresh (**Cmd+Shift+R** or **Ctrl+F5**) to clear stale cache.
3. **Deploy `storage_test.py` First**:
   - Paste the contents of `contracts/storage_test.py`.
   - Click **Deploy**.
   - Test writing a value via `set_value` and reading it back via `get_value`.
   - Confirm that the transaction in the left sidebar shows `Result: SUCCESS` (not just `FINALIZED`).
4. **Deploy `apology_escrow.py`**:
   - Once the sanity test passes, copy the contents of `contracts/apology_escrow.py`.
   - Click **Deploy**.
   - Keep the deployed address handy for testing the scenarios in `test_inputs.md`.

## Studio Error to Rule Mapping Table

If your deployment fails or transactions return errors, locate the message in the table below to fix the issue:

| Error Message / Symptom | Likely Root Cause | Rule to Fix |
| :--- | :--- | :--- |
| `AssertionError: Is right the same storage type? TreeMap <- TreeMap` | Reassigning a `TreeMap` or `DynArray` in `__init__`. | Remove `self.xxx = TreeMap()` from `__init__`. |
| `Contract Queues not found` or endless loading in `IdlenessPhase`/`RevealingPhase` | Missing or incorrect compiler version header comments. | Ensure lines 1 & 2 exactly match the `# v0.2.16` header. |
| `TypeError: unsupported type ...` or payload parse failure | Using `float`, Python native `dict` or `list` in public method signatures. | Convert public inputs/outputs to `str` (JSON strings) or allowed types. |
| `VM Error` during execution of `evaluate_apology` | Calling `gl.nondet.exec_prompt` or `gl.nondet.web.render` directly without `run_nondet_unsafe`. | Wrap all non-deterministic logic in a local `leader_fn` and run it via `run_nondet_unsafe`. |
| Consensus failure / multiple validator retries | Validator checking for exact equality of scores or text generated by LLMs. | Update `validator_fn` to only verify the final `verdict` bucket (e.g. `leader == validator`). |
